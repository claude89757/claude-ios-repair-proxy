Claude iOS 登录卡死修复指南

主要用于账号被 ban、禁用或异常后，Claude iOS 反复出现 “Something went wrong, try again”，删除并重装后仍回不到登录页的场景。按步骤临时连接修复代理，清理旧 session、cookie、routing hint 后立即关闭代理并撤销证书信任。

操作顺序

修复步骤

1

获取邀请码

联系管理员获取本次修复的邀请码。公开页面不内置代理账号密码；验证成功后，页面会显示临时代理配置和证书链接。

2

安装并信任证书

  1. 使用 Safari 打开 证书链接，允许下载描述文件。
  2. 进入 设置 → 通用 → VPN 与设备管理，确认证书描述文件已经安装。
  3. 进入 设置 → 通用 → 关于本机 → 证书信任设置，确认 mitmproxy 证书已经打开“完全信任”。
3

只让 Wi-Fi 走代理

  1. 打开飞行模式，然后只打开 Wi-Fi，保持蜂窝网络关闭。
  2. 进入当前 Wi-Fi 的信息页，找到 HTTP 代理，选择手动。
  3. 填写页面显示的服务器和端口，认证保持关闭，其余认证字段不用填写。
4

打开 Claude

强退 Claude 后重新打开一次。代理会尝试把旧登录态改写为登录过期响应，帮助 App 清理残留并回到登录页。看到登录页或检查项完成后及时停止。

5

恢复网络

关闭 iPhone HTTP 代理，回到证书信任设置中关闭修复 CA 的完全信任。专属端口默认 24 小时失效，不需要长期保留。

    安全说明

    • 请只在自己的设备和账号上使用，修复代理只用于清理卡住的本地旧会话，不用于绕过账号限制。
    • 修复期间不要提交、记录或共享真实 Cookie、sessionKey、routingHint、Authorization、mitmproxy 证书或设备标识。
    • 服务端只记录脱敏状态和事件元数据，不记录 Cookie、请求体或完整设备标识。
    • 完成后关闭 Wi-Fi 代理，并取消修复 CA 的完全信任；专属端口默认 24 小时失效。
    • 如果状态长期没有变化，先确认 iPhone 当前 Wi-Fi、证书信任和代理配置是否一致。
