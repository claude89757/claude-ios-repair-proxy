iPhone / Claude App / 临时修复代理

Claude iOS 登录卡死修复指南

主要用于账号被 ban、禁用或异常后，Claude iOS 反复出现 “Something went wrong, try again”，删除并重装后仍回不到登录页的场景。按步骤临时连接修复代理，清理旧 session、cookie、routing hint 后立即关闭代理并撤销证书信任。

操作顺序

修复向导

按卡片逐步操作。每完成一步点击“我已完成，下一步”，右侧实时状态会同步显示代理连接、证书信任和 Claude 请求事件。

1

准备

获取邀请码

邀请码验证成功。页面已解锁临时代理配置和修复步骤，请继续安装并信任证书。

邀请码已验证 专属代理端口已在右侧状态栏显示。下一步只需要按提示完成证书和 Wi-Fi 代理设置。
2

证书

安装并信任证书

  1. 使用 Safari 打开 证书链接，允许下载描述文件。
  2. 进入 设置 → 通用 → VPN 与设备管理，确认证书描述文件已经安装。
  3. 进入 设置 → 通用 → 关于本机 → 证书信任设置，确认 mitmproxy 证书已经打开“完全信任”。
3

网络

只让 Wi-Fi 走代理

  1. 打开飞行模式，然后只打开 Wi-Fi，保持蜂窝网络关闭。
  2. 关闭手机上的其它 VPN、代理或梯子工具，否则 Claude 流量可能不会走到修复代理。
  3. 进入当前 Wi-Fi 的信息页，找到 HTTP 代理，选择手动。
  4. 填写页面显示的服务器和端口，认证保持关闭，其余认证字段不用填写。
4

修复

打开 Claude

  1. 强退 Claude App，再重新打开。
  2. 等待 10-20 秒，保持当前 Wi-Fi 和代理不变。
  3. 看到登录入口，或状态出现 /api/account、rewrite、Cookie 删除，即可下一步；不需要长时间保持代理。
5

收尾

恢复网络

  1. 把当前 Wi-Fi 的 HTTP 代理改回“关闭”。
  2. 关闭飞行模式，恢复蜂窝网络；恢复你日常使用的 VPN、代理或梯子。
  3. 关闭 mitmproxy 证书“完全信任”。这一步做完后再继续正常使用 Claude。

使用后处理

安全说明

  • 请只在自己的设备和账号上使用，修复代理只用于清理卡住的本地旧会话，不用于绕过账号限制。
  • 修复期间不要提交、记录或共享真实 Cookie、sessionKey、routingHint、Authorization、mitmproxy 证书或设备标识。
  • 服务端只记录脱敏状态和事件元数据，不记录 Cookie、请求体或完整设备标识。
  • 完成后关闭 Wi-Fi 代理，并取消修复 CA 的完全信任；专属端口默认 24 小时失效。
  • 如果状态只显示已连接但没有 Claude 请求，优先检查是否还有其它 VPN、代理或梯子工具未关闭。
  • 如果状态长期没有变化，先确认 iPhone 当前 Wi-Fi、证书信任和代理配置是否一致。
