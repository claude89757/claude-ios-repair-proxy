iPhone / Claude App / 本地登录态修复

Claude iOS 登录卡死修复指南

主要用于账号被 ban、禁用或异常后，Claude iOS 反复出现 “Something went wrong, try again”，删除并重装后仍回不到登录页的场景。按步骤临时配置 Wi‑Fi HTTP 修复通道，帮助 Claude iOS 清理本地旧 session、cookie、routing hint。修复完成后请立即关闭 Wi‑Fi HTTP 代理并撤销证书完全信任。

操作顺序

修复向导

按卡片逐步操作。每完成一步点击“我已完成，下一步”，右侧实时状态会同步显示代理连接、证书信任和 Claude 请求事件。

1

准备

获取邀请码

邀请码验证成功。页面已解锁临时代理配置和修复步骤，请继续安装并信任证书。

邀请码已验证 专属修复端口已在右侧状态栏显示。下一步只需要按提示完成证书和 Wi‑Fi HTTP 代理设置。
2

证书

安装并信任证书

  1. 使用 Safari 打开 证书链接，允许下载描述文件。
  2. 进入 设置 → 通用 → VPN 与设备管理，确认证书描述文件已经安装。
  3. 进入 设置 → 通用 → 关于本机 → 证书信任设置，确认 mitmproxy 证书已经打开“完全信任”。
3

网络

配置 Wi‑Fi 修复通道

  1. 打开飞行模式，然后只打开 Wi-Fi，保持蜂窝网络关闭；关闭手机上的其它 VPN、代理或第三方网络工具，否则 Claude 流量可能不会进入本次修复通道。
  2. 进入 设置 → Wi‑Fi，确认已经连接当前 Wi‑Fi。
  3. 在当前 Wi‑Fi 名称右侧，点击 蓝色圆圈 i / 信息按钮，进入 Wi‑Fi 信息页。
  4. 向下找到 HTTP 代理，选择 手动。
  5. 填写 Header 固定显示的 服务器 和 端口；认证保持关闭，其余认证字段不用填写。
4

修复

打开 Claude

  1. 强退 Claude App，再重新打开。
  2. 等待 10-20 秒，保持当前 Wi‑Fi 和 HTTP 代理设置不变。
  3. 看到登录入口，或状态出现 /api/account、rewrite、Cookie 删除，即可下一步；不需要长时间保持修复通道。
5

收尾

恢复网络

  1. 把当前 Wi-Fi 的 HTTP 代理改回“关闭”。
  2. 关闭飞行模式，恢复蜂窝网络。若你原本有其他网络配置，请自行按原状恢复；本工具不提供 VPN、翻墙、网络加速或地区访问能力。
  3. 关闭 mitmproxy 证书“完全信任”。这一步做完后再继续正常使用 Claude。

使用后处理

安全说明

  • 请只在自己的设备和账号上使用，本工具只用于清理卡住的本地旧会话，不用于绕过账号、地区或网络限制。
  • 修复期间不要提交、记录或共享真实 Cookie、sessionKey、routingHint、Authorization、mitmproxy 证书或设备标识。
  • 服务端只记录脱敏状态和事件元数据，不记录 Cookie、请求体或完整设备标识。
  • 完成后关闭 Wi‑Fi HTTP 代理，并取消修复 CA 的完全信任；公开入口生成的临时端口默认 1 小时失效，售后邀请码以管理员设置为准。
  • 如果状态只显示已连接但没有 Claude 请求，优先检查是否还有其它 VPN、代理或第三方网络工具未关闭。
  • 如果状态长期没有变化，先确认 iPhone 当前 Wi-Fi、证书信任和代理配置是否一致。
