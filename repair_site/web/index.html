操作顺序

修复步骤

1

获取邀请码

联系管理员获取本次修复的邀请码。验证成功后，页面会显示临时代理配置和证书下载入口。

2

安装证书

使用 Safari 打开 /certs/mitmproxy-ca-cert.cer，安装描述文件，并在 iOS 证书信任设置中启用完全信任。

3

设置 Wi-Fi 代理

在当前 Wi-Fi 的 HTTP 代理里选择手动，只填写页面显示的服务器和端口，认证保持关闭。

4

打开 Claude

强退 Claude 后重新打开一次。看到登录页或状态显示已完成后，不要继续长时间保持代理连接。

5

恢复网络

关闭 iPhone HTTP 代理，回到证书信任设置中关闭修复 CA 的完全信任。

    安全说明

    • 修复代理只用于清理卡住的本地旧会话，不用于绕过账号限制。
    • 修复期间不要泄露 Cookie、sessionKey、routingHint、Authorization 或设备标识。
    • 完成后关闭 Wi-Fi 代理，并取消修复 CA 的完全信任。
    • 如果状态长期没有变化，先确认 iPhone 当前 Wi-Fi、证书信任和代理配置是否一致。
